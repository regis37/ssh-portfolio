# ssh-portfolio

> An interactive terminal portfolio you reach over SSH.

```
ssh registsafack.duckdns.org
```

No account, no password, no `-p`. Navigate with `j`/`k`, switch theme with `t`, quit with `q`.

## What this project actually is

This is a **vibecoding experiment**. I wrote **zero lines of code by hand**.
Every line was generated by AI under my direction — the point of the project
was not to learn Go, but to see how far I can get by *orchestrating* AI tools
on a real, deployed, public-facing system.

My role was the engineering direction around the code:
- breaking the goal into phases (server → TUI → content → theming → deploy)
- writing the prompts, reviewing every diff, and steering fixes
- doing everything the AI *can't* do itself: provisioning the cloud VM,
  DNS, firewall rules, SSH keys, and validating the result end-to-end

**Tools used:** Claude (planning & architecture), Claude Code in the terminal
(code generation, git, deployment), ChatGPT (cross-checking), Oracle Cloud
Always Free (hosting), DuckDNS (domain).

## Stack

Go · [Wish](https://github.com/charmbracelet/wish) (SSH server) ·
[Bubble Tea](https://github.com/charmbracelet/bubbletea) (TUI) ·
[Lip Gloss](https://github.com/charmbracelet/lipgloss) (styling) ·
systemd · iptables · deployed on an Oracle Cloud ARM/AMD Always Free VM.

## Controls

| Key       | Action              |
|-----------|---------------------|
| `j` / `↓` | Next section        |
| `k` / `↑` | Previous section    |
| `Enter`   | Open section        |
| `Tab`     | Toggle panel focus  |
| `t`       | Toggle theme        |
| `?`       | Help                |
| `q`       | Quit                |

## Build & run locally

```bash
go mod tidy
go build ./...
PORTFOLIO_ADDR=127.0.0.1:2323 ./portfolio
# then, in another terminal:
ssh -p 2323 127.0.0.1
```
